# 1. Introduction

본 문서는 **'왓 투 웨어(WhatToWear)'** 시스템의 개념화(Conceptualization) 단계 요구사항을 구체화하기 위해 작성된 분석(Analysis) 단계의 산출물이다. 향후 시스템 설계 및 구현의 명확한 기준선을 제공하기 위해, 본 시스템을 구성하는 핵심 데이터 객체의 정적 구조를 정의하는 도메인 모델(Domain models), 사용자와 시스템 간의 동적 상호작용 흐름을 기술하는 유스케이스 명세서(Use case specifications), 그리고 주요 기능의 시간적 메시지 전달 과정을 시각화한 시스템 시퀀스 다이어그램(System sequence diagram)을 명세한다.

<br>

## 1.1 System Summary (시스템 요약)
'왓 투 웨어'는 실시간 기상 데이터에 사용자 고유의 체질(열 민감도)과 외출 목적(TPO)을 결합하여 개인화된 옷차림을 추천하는 지능형 맞춤형 서비스이다. 기존 일기예보가 해결하지 못하는 주관적인 체감 온도의 차이를 데이터화하여 일상 속 옷차림에 대한 고민과 불편함을 근본적으로 해소하는 것을 목표로 한다.

<br>

## 1.2 Prominent Features (프로젝트 주요 특징)

### ✅ 유용성 (Usefulness)
객관적인 기온 수치만 제공하는 기존 일기예보의 한계를 극복하고, 사용자의 주관적인 체감 온도를 계산하여 상황에 맞는 최적의 옷차림을 즉각적으로 제시함으로써 매일 아침의 고민과 외출 시의 불편함을 해소한다.

### ✅ 중요성 및 의의 (Significance)
외출 후 사용자가 남긴 체감 온도 피드백을 시스템이 정량적 데이터로 수집하고, 추천 알고리즘의 가중치를 스스로 업데이트(자가 학습)하여 사용할수록 개인에게 완벽히 최적화되는 경험을 제공한다.

### ✅ 확장성 (Expandability)
개인 옷장 데이터를 활용하는 것을 넘어, 향후 외부 의류 커머스 API와 연동한 의류 추천 및 판매로 확장할 수 있다. 또한 미세먼지나 강수량 등 복합 기상 데이터를 추가 연동하여 알고리즘을 고도화할 수 있는 유연한 구조를 가진다.

# 3. Domain analysis

본 프로젝트의 유스케이스 명세서를 바탕으로 시스템을 구성하는 핵심 데이터 객체(Class)를 도출하고 그 의미를 정의한다.

| Class Name (클래스명) | Description (의미 및 역할) |
| :--- | :--- |
| **`User`** | 서비스를 이용하는 주체. 사용자의 기본 계정 정보(ID, 비밀번호, 닉네임)와 시스템 인증 상태를 관리하는 클래스이다. |
| **`UserProfile`** | 사용자의 고유한 체질(온도 민감도) 데이터를 나타내는 클래스. 5단계의 체질 입력값을 바탕으로 계산된 '열 민감도 가중치(Weight)'를 저장하고 갱신한다. |
| **`Clothes`** | 사용자가 디지털 옷장에 등록한 개별 의류 데이터를 담는 클래스. 옷의 식별용 이름, 카테고리(상의/하의/아우터), 두께(계절감), 그리고 상황 태그(TPO) 정보를 포함한다. |
| **`WeatherData`** | 외부 기상청 API로부터 수신하는 실시간 날씨 객체. 추천 알고리즘의 기준이 되는 현재 기온, 강수 확률, 풍속 등의 기상 정보를 담고 있으며, 통신 실패 시 사용자의 수동 입력값도 이 객체에서 처리된다. |
| **`OutfitGuide`** | 날씨, 체질 가중치, TPO 조건이 모두 연산되어 최종적으로 도출된 '옷차림 추천 결과물'을 담는 클래스. 사용자 화면에 출력될 상의/하의/아우터 조합 데이터와 기상 알림(우산 등) 메시지를 포함한다. |
| **`Feedback`** | 추천받은 가이드에 대해 사용자가 외출 후 입력하는 체감 온도 평가 데이터(5단계). 이 데이터는 일회성으로 수집되어 `UserProfile`의 가중치를 재학습시키는 데 사용된다. |
