## 1. Business purpose

**1.1 Project Background**

"오늘 쌀쌀하니까 겉옷 챙겨 입어!"라는 주변의 조언이나 날씨 앱의 기온만 믿고 외출했다가 낭패를 본 경험은 누구나 한 번쯤 겪어보았을 일입니다. 언론 보도에 따르면, 낮과 밤의 일교차가 15도 이상 크게 벌어지거나 갑작스러운 기온 변화가 찾아오면 포털 사이트에서 '기온별 옷차림' 검색량이 평소 대비 4배 이상 급증할 정도로 날씨에 맞는 옷차림을 결정하는 것은 대중들의 보편적인 고민거리입니다. 이를 해결하기 위해 통계청 등에서 12~16도에는 카디건, 9~11도에는 트렌치코트와 같은 표준 가이드를 제공하고 있지만, 이 역시 한계가 존재합니다. 

본 프로젝트는 이처럼 통계적인 기준이나 남들의 조언에만 맞춰 옷을 챙겨 입었다가, 평소 땀이 많고 더위를 잘 타는 등 '개인의 고유한 체질'을 간과하는 바람에 하루 종일 극심한 불쾌감을 겪어야 했던 실제 경험에서 출발했습니다. 기존의 서비스들은 "오늘 기온이 20도다"라는 객관적 수치나 보편적인 기준만 제공할 뿐, 사용자 개개인이 더위나 추위를 느끼는 '주관적인 체감의 차이'는 전혀 반영하지 못한다는 명확한 문제점을 지니고 있습니다. 따라서 본 프로젝트는 이러한 한계를 극복하고, 개인의 체질을 바탕으로 개인별 체감 온도를 계산하여 일상 속 불편함을 근본적으로 해소해 주는 지능형 맞춤형 옷차림 가이드 서비스를 구현하고자 합니다.

**1.2 Goals**

- **맞춤형 알고리즘 엔진 개발:** 사용자의 체질(열 민감도)을 데이터화하여 맞춤형 옷차림을 추천해 주는 시스템을 구축하는 것이 1차 목표입니다. 기상청 API 기온에 사용자의 체질 데이터를 더해 나만의 체감 온도를 계산하고 최적의 옷차림을 추천합니다.
- **TPO(상황/목적) 기반 맞춤형 필터링:** 단순히 온도만 고려하는 것을 넘어, 사용자의 당일 외출 목적을 입력받아 상황에 부적절한 의류를 배제하고 가점을 부여합니다.
- **피드백 기반 자가 학습 구현:** 외출 후 웹 사이트에서 제공되는 체감 온도 피드백 버튼(예: 추웠다, 조금 쌀쌀했다, 적당했다, 따뜻했다, 더웠다) 중 하나를 간편하게 선택하면, 시스템이 이를 정량적인 데이터로 변환하여 다음 추천 알고리즘의 가중치를 스스로 조절해 점점 완벽한 가이드로 진화하도록 만듭니다.
- **사용자 경험 극대화:** 매일 아침 "오늘 날씨에 뭐 입지?" 고민하는 시간을 획기적으로 줄이고, 사용자가 가진 옷들을 속성별로 등록해 스마트하게 관리할 수 있는 환경을 제공합니다.

**1.3 Target Market**

- **체질적 특성이 뚜렷한 사람:** 일반적인 기준보다 땀이 많아 더위를 크게 타거나 반대로 남들보다 추위를 유독 많이 타는 등 고유의 열 민감도를 가지고 있어, 기존의 일기예보만으로는 외출복을 결정하기 애매한 사람들
- **온도 감각이 무뎌 매번 옷차림을 고민하는 사람들:** 기온만 보고는 도저히 어떤 두께의 옷을 입어야 할지 감을 못 잡는 사람들, 덥다고 겉옷을 하루 종일 짐처럼 들고 다니는 등 시행착오를 반복하며 데이터 기반의 확실한 옷차림을 원하는 사람들

## 4. Concept of operation

**1. Sign Up**

| 구분 | 내용 |
|:---|:---|
| **Purpose** |이 시스템을 사용하기 위해 사용자를 등록하여 서비스 이용 권한 부여. |
| **Approach** |사용자가 회원가입 화면에서 필요한 정보를 입력하고 가입을 요청하면, 시스템이 서버에 새 회원 정보를 생성한다.|
| **Dynamics** |웹 사이트 최초 접속 및 신규 계정 생성이 필요할 경우.|
| **Goals** |회원가입 기능을 구현한다.|

**2. Log In**

| 구분 | 내용 |
|:---|:---|
| **Purpose** |웹 서비스를 사용하기 위해 등록된 사용자인지 확인. |
| **Approach** |사용자가 웹 사이트 접속 후 로그인시, ID, PW를 입력 후 로그인을 요청하면 서버에서 회원 정보를 조회 후 로그인을 진행한다.|
| **Dynamics** |웹 사이트 접속 시 로그인할 경우.|
| **Goals** |로그인 기능을 구현한다.|

**3. Log Out**

| 구분 | 내용 |
|:---|:---|
| **Purpose** |사용자가 서비스를 그만 이용하고 싶을 경우. |
| **Approach** |사용자가 웹 사이트 내에서 로그아웃을 요청하면, 서버에서 사용자의 접속 세션을 만료시키고 초기 화면으로 이동시킨다.|
| **Dynamics** |웹 서비스 이용을 마치고 계정 접속을 안전하게 종료하고자 할 경우.|
| **Goals** |로그아웃 기능을 구현한다.|

**4. Set Profile**

| 구분 | 내용 |
|:---|:---|
| **Purpose** |사용자 맞춤형 알고리즘을 위한 초기 열 민감도 데이터 확보. |
| **Approach** |사용자가 프로필 설정 화면에서 본인의 추위/더위 타는 정도를 선택하여 제출하면 시스템이 이 데이터를 수집한다.|
| **Dynamics** |회원가입 직후 또는 웹 사이트 내 설정 메뉴에서 프로필을 초기화/수정할 경우.|
| **Goals** |초기 체질 프로필 설정 기능을 구현한다.|

**5. Add Clothes**

| 구분 | 내용 |
|:---|:---|
| **Purpose** |사용자 상황(TPO)에 맞는 최적의 옷차림 추천 결과 도출 시작.|
| **Approach** |사용자가 웹의 디지털 옷장 화면에서 옷의 종류, 두께, TPO 속성 등을 선택하고 등록을 요청하면 시스템이 이 데이터를 수집한다.|
| **Dynamics** |사용자가 새로운 옷을 시스템에 등록하고자 할 경우.|
| **Goals** |소유 의류 등록(디지털 옷장) 기능을 구현한다.|

**6. Request Guide**

| 구분 | 내용 |
|:---|:---|
| **Purpose** |사용자 상황(TPO)에 맞는 최적의 옷차림 추천 결과 도출 시작. |
| **Approach** |사용자가 외출 목적(TPO)을 선택하고 가이드를 요청하면, 시스템이 날씨 데이터와 사용자 정보를 기반으로 추천 알고리즘 연산을 시작한다.|
| **Dynamics** |사용자가 외출 전 맞춤형 옷차림 추천을 받기 위해 웹에서 버튼을 클릭할 경우.|
| **Goals** |맞춤형 옷차림 추천 요청 기능을 구현한다.|

**7. Send Feedback**

| 구분 | 내용 |
|:---|:---|
| **Purpose** |추천 알고리즘의 정확도 향상을 위한 자가 학습 데이터 수집. |
| **Approach** |외출 후 돌아온 사용자가 웹 사이트를 통해 당일 추천받은 옷차림에 대한 체감온도(더움/추움 등)를 선택하여 전송한다.|
| **Dynamics** |외출 일정이 종료된 후 사용자가 웹의 피드백 메뉴에 진입할 경우.|
| **Goals** |체감 온도 피드백 전송 기능을 구현한다.|

**8. Show Guide**

| 구분 | 내용 |
|:---|:---|
| **Purpose** |산출된 최적의 옷차림 조합을 사용자에게 시각적으로 전달. |
| **Approach** |가이드 엔진이 연산을 완료하면 조합된 옷차림 결과와 예상 체감 온도를 웹 브라우저 UI를 통해 사용자 화면에 출력한다.|
| **Dynamics** |시스템이 옷차림 추천 알고리즘 연산을 완료한 직후.|
| **Goals** |가이드 결과 제공 화면을 구현한다.|

**9. Get Weather Data**

| 구분 | 내용 |
|:---|:---|
| **Purpose** |알고리즘 연산에 필요한 실시간 외부 기상 정보 확보. |
| **Approach** |사용자가 가이드를 요청하면, 시스템이 사용자의 위치 정보를 바탕으로 외부 기상청 API에 기상 데이터를 요청한다.|
| **Dynamics** |사용자의 옷차림 추천 가이드 요청(Request Guide)이 발생한 직후.|
| **Goals** |외부 기상 API 데이터 요청 기능을 구현한다.|

**10. Return Weather Data**

| 구분 | 내용 |
|:---|:---|
| **Purpose** |시스템의 요청에 대한 기상 데이터 수신. |
| **Approach** |외부 기상 API가 시스템의 요청을 받아 해당 지역의 실시간 날씨 데이터(기온, 풍속 등)를 시스템에 반환한다.|
| **Dynamics** |기상 API가 시스템의 데이터 요청을 성공적으로 처리했을 경우.|
| **Goals** |외부 기상 데이터 수신 기능을 구현한다.|

**11. Save Profile**

| 구분 | 내용 |
|:---|:---|
| **Purpose** |수집된 사용자 프로필 및 의류 데이터의 영구 저장. |
| **Approach** |사용자가 입력한 체질 정보를 시스템이 Database에 Insert/Update 쿼리를 통해 저장한다.|
| **Dynamics** |사용자가 프로필 설정이나 옷 등록을 완료하여 제출할 경우.|
| **Goals** |데이터베이스 정보 저장 기능을 구현한다.|

**12. Update Weight**

| 구분 | 내용 |
|:---|:---|
| **Purpose** |피드백을 반영한 알고리즘 고도화(자가 학습) 적용. |
| **Approach** |시스템이 사용자의 피드백을 받아 기존 체질 가중치 공식을 재계산하고, 변경된 값을 Database에 갱신(Update)한다.|
| **Dynamics** |사용자가 체감 온도 피드백을 시스템에 전송했을 경우.|
| **Goals** |피드백 기반 체질 가중치 업데이트 기능을 구현한다.|

**13. Load Profile**

| 구분 | 내용 |

## 5. Problem statement

**5.1 Overview**

본 프로젝트는 사용자의 고유한 체질과 외출 목적을 분석하여 최적의 옷차림을 추천해 주는 맞춤형 웹 서비스입니다. 단순히 기상청 날씨 정보의 전달을 넘어, 사용자의 피드백을 통해 나만의 가중치를 스스로 학습하는 지능형 가이드 엔진이 핵심입니다. 따라서 이 복잡한 추천 알고리즘을 구현하는 과정에서 예상되는 기술적 난제들과, 시스템이 사용자에게 불편함 없이 안정적으로 제공되어야 할 기준을 아래와 같이 정의합니다.

**5.2 Problems**
* **주관적인 체감 온도의 수치화:** "덥다", "춥다"라는 사용자 개개인의 주관적인 감각을 컴퓨터가 계산할 수 있는 정량적인 수치(가중치)로 변환하는 것이 가장 큰 기술적 난제이다. 사용자의 피드백을 기반으로 알고리즘 공식을 스스로 업데이트하는 자가 학습 로직을 설계해야 한다.
* **초기 데이터 부족으로 인한 추천 정확도 저하:** 서비스 초기에는 피드백 데이터가 누적되지 않아 맞춤형 추천의 정확도가 일시적으로 떨어질 수 있다. 이를 방지하기 위해 가입 시 초기 체질 프로필 설정을 세분화하는 방안을 고려해야 한다.
* **외부 API 의존성에 따른 단일 장애점 위험:** 핵심 기능이 외부 '기상청 API 날씨 데이터'에 크게 의존하므로, 만약 기상청 서버에 장애가 발생하거나 네트워크 지연이 생길 경우 우리 시스템의 추천 엔진까지 멈춰버리는 문제가 발생할 수 있어 이에 대한 기술적 대비책이 필요하다.

**5.3 NFRs**
1) 사용자가 '가이드 추천' 버튼을 눌렀을 때, 외부 데이터 수신 및 알고리즘 연산 과정을 포함하여 모든 결과 출력이 **최대 2초 이내**에 완료되어 지연 없는 사용자 경험을 제공해야 한다.
2) 날씨 API가 응답하지 않는 상황(Time-out)이 발생하더라도 시스템이 다운되지 않아야 한다. 이 경우 캐싱된 최근 데이터를 띄워주거나 기본 예보 데이터를 보여주는 예외 처리(Fallback) 로직이 작동해야 한다.
3) 아침 출근/통학 시간 등 바쁜 상황에서 스마트폰으로 빠르게 접근할 수 있도록, 접속하는 기기의 화면 크기에 맞춰 레이아웃이 최적화되는 **모바일 반응형 웹(Responsive Web)** 인터페이스로 구축해야 한다.
4) 향후 의류 카테고리나 TPO 속성이 새롭게 추가될 것을 대비하여, 코드 구조를 유연하게 확장할 수 있도록 설계해야 한다.

## 6. Glossary

| 용어 | 설명 |
|:---|:---|
| **TPO** | 옷을 입을 때 고려해야 하는 시간, 장소, 상황/목적을 의미하는 용어. 본 문서에서는 사용자의 당일 외출 목적을 지칭하며, 시스템이 의류를 필터링하는 핵심 기준값으로 사용된다. |
| **가이드 엔진** | 기상청의 날씨 데이터, 사용자의 체질 가중치(열 민감도), TPO, 사용자의 옷장 데이터를 종합적으로 연산하여 최적의 옷차림 조합을 산출해 내는 본 시스템 내부의 핵심 알고리즘 모듈이다. |
| **열 민감도** | 사용자가 일반적인 기준에 비해 추위나 더위를 얼마나 타는지를 수치화한 데이터. 초기 프로필 설정 및 피드백 자가 학습을 통해 지속적으로 업데이트되는 사용자 고유의 가중치 변수이다. |
| **모바일 반응형 웹** | PC, 태블릿, 스마트폰 등 사용자가 접속하는 기기의 디스플레이 화면 크기에 맞춰 웹 페이지의 레이아웃과 UI가 자동으로 최적화되어 변환되는 웹 개발 기법이다. |
| **API** | 서로 다른 소프트웨어 시스템이 데이터를 안전하게 주고받을 수 있도록 만들어진 통신 창구. 본 시스템에서는 실시간 기상 데이터를 수신하기 위해 외부 'Weather API'를 활용한다. |
| **필터링 및 적합도 평가** | 추천 알고리즘의 핵심 연산 과정. 사용자가 입력한 TPO에 부적절한 옷은 후보군에서 배제하고, 상황에 적합한 옷에는 가산점을 부여하여 최종 추천 순위를 매기는 방식이다. |
